Core Customization Topics

 

The ScriptEase core is highly customizable to suit the needs of a variety of scripted applications. This chapter described the customization options available. All of these options are determined by compile-time #defines. Once the options are set, the ScriptEase core must be recompiled to reflect these options. As a result, only customers who have purchased ScriptEase and thus have the source code to the ScriptEase core can benefit from customizing the core.

 

The jseopt.h file found in the distribution contains the same documentation as is here. For each define, it starts either on or off (off being commented out). Read through this chapter and the jseopt.h file and change the state of any of the options you like. Below, each option is listed as either on or off, reflecting the default state.
